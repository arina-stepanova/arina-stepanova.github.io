How to understand: You can set your own text color for any element of a web page. The color property sets it for text, as well as for its design elements, such as underlining, lines above text, strikethroughs, and others. To set the background of text, use the background-color property, and the borders of an element can be colored with border-color.

KEYWORD VALUE

transparent

The transparent keyword specifies a transparent text color. This is technically equivalent to writing any color with zero transparency rgb(0 0 0 / 0), but there are times when just transparency and color transparency can work differently, for example in gradients.

currentColor

The currentColor keyword hides the color specified in the element's color property. It is very convenient to specify this word as the value of other properties in order not to repeatedly specify the same color.

HEX

Hexadecimal color code in the RGB color model that starts with #, for example, #ff0000. Solid colors are written in the format #RRGGBB or abbreviated #RGB (if the characters of each group are the same). For example #009900 or #090. If transparency needs to be specified, it is added at the end in the #RRGGBBAA or #RGBA HEX format, such as #00990055 or #0905.

Previously, it was impossible to set a color in #RGBA notation and you had to use the rgba() function, but today this method has good cross-browser compatibility. There is only one problem: few people can immediately calculate 50% in hexadecimal format, therefore, to specify the transparency of a color, the rgb() function is most often used, where transparency can be set in fractions or percentages.

RGB

The rgb() function is used to set the color, for example rgb(0 63 255) for blue. Each of the three values is responsible for a separate RGB channel and can be written as a number from 0 to 255 or as a percentage. To add transparency, after recording the channels, put a slash and write the desired value from 0 to 1 or as a percentage, for example, rgb(0 63 255 / 0.5) for translucent blue.

Previously, the RGB syntax was different from the modern one, and you can still find it in the code or choose it for better cross-browser compatibility, see Can I use. To separate channels inside the function, it was necessary to use commas rgb(0, 63, 255), and to add transparency, a special function rgba(), which took the transparency of the color as the last parameter, for example rgba(255, 0, 0, 0.5) .

HSL

The HSL color model describes the same colors as RGB, but differently:

  • H - Hue
  • S - Saturation
  • L - Lightness
For example, hsl(120 100% 50%) for green. The first hue value is given in degrees and can be written simply as 120 (as is most often done) or with rotation units: deg, turn, grad, rad. Values are available from 0 to 360 if you specify rotation in degrees. Negative values are not allowed. The second and third values are given as a percentage. Transparency is added in the same way as in rgb, using a slash with a value like hsl(120 100% 50% / 0.5) translucent green.

The hsl() syntax differed from the modern one in the same way as rgb(): it needed commas and a special hsla() function to set the transparency of the color. Use the old syntax for better compatibility and don't be surprised if you see it in your code.

HWB

  • H - hue
  • W - whiteness (amount of white)
  • B - blackness (amount of black)
The first hue value is given in degrees and can be written simply as 120 (as is most often done) or with rotation units: deg, turn, grad, rad. Values are available from 0 to 360 if you specify rotation in degrees. The second and third values are given as a percentage. It is permissible to specify color transparency as the fourth parameter through a slash.

If the second and third parameters add up to 100%, then they are normalized to 100% automatically.
